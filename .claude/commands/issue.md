基于用户的描述，在当前仓库创建一个 GitHub Issue。

## 工作流程

1. **理解问题**：分析用户描述的问题或需求
2. **阅读代码**：阅读相关源码，深入理解问题的技术细节和影响范围
3. **创建 Issue**：使用 `gh issue create` 提交 issue

## Issue 格式要求

- 标题：简洁明确，用中文描述核心问题
- 正文结构：
  - `## 问题描述`：说明当前行为和问题所在，包含相关代码路径和逻辑分析
  - `### 复现场景`（如果是 bug）：给出具体的操作步骤
  - `### 影响范围`：列出受影响的模块和文件
  - `## 建议方案`：给出一个或多个可行的解决思路，包含需要考虑的技术细节
  - `## 标签建议`：推荐合适的标签

## 注意事项

- 所有描述使用中文
- 分析要基于实际代码，不要臆测
- 影响范围要列出具体文件路径
- 方案要具体可执行，不要泛泛而谈
